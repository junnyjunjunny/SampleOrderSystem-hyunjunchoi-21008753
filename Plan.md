# Plan: 콘솔 테이블 서식 개선 — 정렬 + 테두리 + 헤더 강조색

> CLAUDE.md의 "콘솔 UX" 컨벤션에 이미 반영됨: 표는 공용 `print_table()` 헬퍼로 그리고, 단선 박스 문자로 테두리를, 헤더 행은 배경색으로 강조한다.

## 목표

1. 기존에 각 화면마다 제각각이던 표 출력(f-string 수동 패딩)을 공용 헬퍼로 통일한다.
2. 컬럼 폭에 맞춰 tab 정렬 대신 고정폭 정렬 + 테두리를 적용해 가독성을 높인다.
3. 헤더 행에 배경색을 넣어 구분을 명확히 한다.
4. Windows 콘솔(cp949)에서 깨지지 않는 문자만 사용한다.

## 접근 방식

- `app/cli.py`에 `print_table(headers, rows, widths)` 공용 함수를 추가한다.
  - 테두리: 단선 박스 문자(`┌┬┐├┼┤└┴┘─│`) 사용 — cp949 인코딩 가능 확인 완료(이중선 `═║╔` 등은 cp949에서 크래시하므로 제외).
  - 헤더 행: 배경색(예: 파란 배경 + 흰 글씨, ANSI `\033[44m\033[97m ... \033[0m`)으로 강조.
  - 셀 값에 이미 ANSI 색상 코드가 포함된 경우(예: 재고량 확인의 여유/부족/고갈 라벨)는 이중으로 폭 계산하지 않도록 처리 — 호출 측에서 미리 시각적 폭에 맞게 패딩한 문자열을 넘기면 그대로 배치한다.
- 기존 4개 표 출력 함수를 `print_table()` 사용으로 교체:
  - `print_sample_table` (시료 관리/검색)
  - `print_order_approval_table` (주문 승인/거절)
  - `check_order_volume`의 목록 표 (모니터링 - 주문량 확인)
  - `check_stock_level`의 표 (모니터링 - 재고량 확인, 셀 색상 유지)
