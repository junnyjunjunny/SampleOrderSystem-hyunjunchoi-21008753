# 3. 주문 승인/거절

## 개요

2번(시료 주문) 메뉴에서 접수되어 `RECEIVED` 상태로 저장된 주문을, 주문번호를 입력해 승인(`APPROVED`)하거나 거절(`REJECTED`)하는 기능.
승인/거절 판단은 오직 이 메뉴에서만 이뤄진다(2번 메뉴는 접수만 하고 끝남). 승인 시 재고 충분 여부를 검증한다(재고는 이 시점에 차감하지 않고, 출고 시점에 차감).

## 현재 구현 상태

- 승인: 구현됨 (`approve_order` → `OrderService.approve`)
- 거절: 구현됨 (`reject_order` → `OrderService.reject`)

## 상세 요구사항

### 입력값
| 필드 | 타입 | 제약 |
|---|---|---|
| order_id | str | 존재해야 함, 현재 상태가 `RECEIVED`여야 함 |

### 정상 흐름 (승인)
1. 주문을 조회하고, 연결된 시료의 재고(`stock`)가 주문 수량 이상인지 확인한다.
2. 충분하면 상태를 `APPROVED`로 전이한다. (재고 차감은 하지 않음 — 출고 시 차감)

### 정상 흐름 (거절)
1. 주문 상태를 `RECEIVED` → `REJECTED`로 전이한다.

### 예외 케이스
- 존재하지 않는 `order_id` → `KeyError`
- `RECEIVED`가 아닌 주문을 승인/거절 시도 → `ValueError` (`ALLOWED_TRANSITIONS`에 없는 전이)
- 재고가 주문 수량보다 적은데 승인 시도 → `ValueError` — 이 경우 주문은 `RECEIVED`로 남아 나중에 다시 승인/거절을 시도할 수 있다
- 거절된(`REJECTED`) 주문을 재승인하는 것은 허용하지 않음(현재 구조상 자연히 막힘, 기존 테스트로 확인됨)

## 데이터/상태

상태 전이 규칙: `RECEIVED → {APPROVED, REJECTED}`. 자세한 흐름도는 [PRD.md](../PRD.md) 참고.
