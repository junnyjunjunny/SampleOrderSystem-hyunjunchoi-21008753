# 1. 시료 관리

## 개요

시료(Sample)를 등록/조회/검색/수정/삭제하는 기능. 다른 모든 메뉴(주문, 승인, 출고 등)가 참조하는 기준 데이터를 관리한다.
메인 메뉴에서 `[1] 시료 관리`로 진입하면 하위 메뉴(서브메뉴)로 들어간다.

## 현재 구현 상태

- 진입 시 등록 시료 목록 페이지네이션 조회: 구현됨 (`sample_management_menu`)
- 검색(ID/이름 대소문자 무시 부분일치): 구현됨, 하위 메뉴 `[1] 검색`
- 등록: 구현됨, 하위 메뉴 `[2] 등록`
- 수정/삭제: 구현됨, 하위 메뉴 `[3] 수정/삭제` (`edit_or_delete_sample`)

## 하위 메뉴 구조

`[1] 시료 관리` 진입 시:
1. 화면을 지운다(`cls`).
2. 등록된 시료 목록을 5개씩 페이지네이션해서 보여준다(기본 1페이지).
3. 한 줄짜리 통합 프롬프트로 입력을 받는다: `[N] 다음 [B] 이전 [1] 검색 [2] 등록 [3] 수정/삭제 [0] 뒤로 > `

| 입력 | 동작 |
|---|---|
| `N` | 다음 페이지로 이동(마지막 페이지면 이동하지 않고 안내 메시지) |
| `B` | 이전 페이지로 이동(첫 페이지면 이동하지 않고 안내 메시지) |
| `1` | 검색 실행(`search_samples`) — 실행 후 Enter로 목록 화면 복귀 |
| `2` | 등록 실행(`register_sample`) — 실행 후 Enter로 목록 화면 복귀 |
| `3` | 수정/삭제 실행(아래 참고) — 실행 후 Enter로 목록 화면 복귀 |
| `0` | 메인 메뉴로 복귀 |

액션(검색/등록/수정/삭제) 실행 후에는 결과를 보여준 채로 "계속하려면 Enter"로 대기했다가, 목록 화면(1페이지)으로 돌아간다.

### 수정/삭제 (`[3]`)

1. 시료 ID를 입력받는다.
2. 해당 ID가 없으면 `KeyError` → 이 하위 메뉴 루프에서 캐치해 오류 메시지로 표시.
3. 있으면 현재 값을 보여주고 `[1] 수정 [2] 삭제`를 선택받는다.
   - 수정: `name`/`avg_production_time`/`yield_rate`/`stock`을 다시 입력받아(엔터 시 기존 값 유지) 새 `Sample`을 만들고(생성 시점 검증 재사용) `SampleRepository.update`로 반영한다. `sample_id`는 변경 불가.
   - 삭제: `SampleRepository.delete(sample_id)`로 제거.

## 상세 요구사항

### 입력값 (등록)
| 필드 | 타입 | 제약 |
|---|---|---|
| sample_id | str | 필수, 중복 불가 |
| name | str | 필수 |
| avg_production_time | float | 0보다 커야 함 |
| yield_rate | float | 0~1 |
| stock | int | 0 이상, 미입력 시 기본값 0 |

### 정상 흐름 (등록)
1. 사용자가 시료 ID/이름/생산시간/수율/재고를 입력한다.
2. `Sample` 생성 시점(`__post_init__`)에 값 검증을 거친 뒤 `SampleRepository.create`로 저장한다.

### 예외 케이스
- 이미 존재하는 `sample_id`로 등록 시 → `ValueError`
- 존재하지 않는 `sample_id` 조회/수정/삭제 시 → `KeyError`
- `avg_production_time`이 0 이하 → `ValueError` (`Sample.__post_init__`)
- `yield_rate`가 0~1 범위를 벗어남 → `ValueError` (`Sample.__post_init__`)
- `stock`이 음수 → `ValueError` (`Sample.__post_init__`)
- 수정 시에도 동일한 `Sample.__post_init__` 검증이 적용된다(새 값으로 `Sample`을 다시 생성하므로).

### 검색
- 검색 기준(`id` 또는 `name`)을 선택하고 검색어를 입력하면, 대소문자 무시 부분일치(SQL `LIKE ... COLLATE NOCASE`)로 조회한다.
- 검색어에 포함된 `%`/`_`/`\`는 리터럴로 취급하도록 이스케이프 처리한다(`_escape_like`).
- 일치하는 결과가 없으면 빈 목록을 반환한다.
- 빈 문자열로 검색하면 전체 목록을 반환한다.
- `id`/`name` 외의 검색 기준을 넘기면 `KeyError`(이 메뉴 루프에서 캐치되어 오류 메시지로 표시됨).

### 페이지네이션
- 페이지 크기 5, 진입 시 항상 1페이지(0-indexed로는 0페이지)부터 시작.
- 페이지 번호는 유효 범위로 클램프한다(마지막 페이지에서 `N` 눌러도 더 넘어가지 않음, 첫 페이지에서 `B` 눌러도 더 앞으로 가지 않음).
