# 6. 출고 처리

## 개요

메인 메뉴 `[6] 출고 처리`. 승인 완료된(`CONFIRMED`) 주문을 출고(`RELEASE`) 상태로 바꾸는 기능.
**재고는 여기서 차감하지 않는다** — 재고 차감은 3번(주문 승인/거절)에서 즉시 `CONFIRMED`가 될 때, 또는 5번(생산 라인 조회)에서 생산이 완료되어 `CONFIRMED`가 될 때 이미 반영되어 있다. 출고는 순수한 상태 전이(기록)일 뿐이다.

## 현재 구현 상태

- 목록 조회 + 번호 선택 출고: 구현됨 (`release_menu`, `print_release_table` → `OrderService.ship`)

## 하위 메뉴 구조

`[6] 출고 처리` 진입 시:
1. 화면을 지운다(`cls`).
2. `CONFIRMED` 상태 주문 목록을 5개씩 페이지네이션해서 보여준다(시료 관리/주문 승인 화면과 동일한 UX).
   컬럼: **번호(현재 페이지 내 1부터) / 주문번호 / 고객 / 시료 / 수량**
3. 통합 프롬프트: `[N] 다음  [B] 이전  번호 입력(출고)  [0] 뒤로 > `
   - 번호를 입력하면 곧바로 해당 주문을 출고 처리한다(승인/거절과 달리 Y/N 재확인 없음).
   - `0`: 메인 메뉴로 복귀

출고 후에는 결과를 보여준 채로 "계속하려면 Enter"로 대기했다가 목록 화면(1페이지)으로 돌아간다. 방금 출고한 주문은 더 이상 `CONFIRMED`가 아니므로 목록에서 사라진다.

## 상세 요구사항

### 입력값
| 필드 | 타입 | 제약 |
|---|---|---|
| 번호 | int | 현재 페이지에 표시된 항목 범위(1~페이지 항목 수) 안이어야 함 |

### 정상 흐름
1. 주문을 조회한다.
2. 상태를 `CONFIRMED` → `RELEASE`로 전이한다. 재고 변화 없음.

### 예외 케이스
- 페이지 범위를 벗어난 번호 입력 → `ValueError`
- `CONFIRMED`가 아닌 주문(구조적으로 발생 안 함 — 목록에 `CONFIRMED`만 나타나므로 선택 자체가 불가능)

## 재고 차감 시점 (중요)

이전에는 출고 시점에 `stock -= quantity`를 했으나, 이제는 **주문이 `CONFIRMED`가 되는 시점**에 차감이 끝나 있다:
- `OrderService.decide()`: 재고가 충분해 즉시 `CONFIRMED`가 되면, 그 즉시 `stock -= quantity`.
- `OrderService.complete_production()`: `PRODUCING → CONFIRMED`(생산 완료)가 되면, `stock += production_quantity - quantity`(부족분 보충과 이 주문 소비분 차감을 한 번에 반영).
- `OrderService.ship()`: 상태 전이만 하고 재고는 전혀 건드리지 않는다.
