# 6. 출고 처리

## 개요

생산 중(`IN_PRODUCTION`)인 주문을 출고(`SHIPPED`) 처리하는 기능. 출고 시점에 시료 재고를 주문 수량만큼 차감한다.

## 현재 구현 상태

- 출고 처리: 구현됨 (`ship_order` → `OrderService.ship`)

## 상세 요구사항

### 입력값
| 필드 | 타입 | 제약 |
|---|---|---|
| order_id | str | 존재해야 함, 현재 상태가 `IN_PRODUCTION`이어야 함 |

### 정상 흐름
1. 주문을 조회하고 연결된 시료를 조회한다.
2. 상태를 `IN_PRODUCTION` → `SHIPPED`로 전이한다.
3. 시료 재고를 주문 수량만큼 차감한다(`stock -= quantity`).

### 예외 케이스
- 존재하지 않는 `order_id` → `KeyError`
- `IN_PRODUCTION`이 아닌 주문(`RECEIVED`/`APPROVED`/`REJECTED`/`SHIPPED`)을 출고 시도 → `ValueError`
- (TODO) 재고보다 많은 수량이 이미 승인되어 있었던 경우(승인 이후 다른 경로로 재고가 줄어든 경우) 출고 시점에 재고가 음수가 될 수 있는지 → 현재 `update_stock`은 검증 없이 그대로 차감. 재고 음수 방지 로직 필요 여부 논의.
