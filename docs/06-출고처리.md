# 6. 출고 처리

## 개요

메인 메뉴 `[6] 출고 처리`(이전 5번 자리에서 이동). 승인 완료된(`CONFIRMED`) 주문을 출고(`RELEASE`) 처리하는 기능. 출고 시점에 시료 재고를 주문 수량만큼 차감한다.
`CONFIRMED`는 3번(주문 승인/거절)에서 재고가 충분해 바로 확정됐거나, 5번(생산 라인 조회 — 향후 자동 생산 완료 시)에서 부족분을 채우고 확정된 경우 둘 다 해당한다.

## 현재 구현 상태

- 출고 처리: 구현됨 (`ship_order` → `OrderService.ship`)

## 상세 요구사항

### 입력값
| 필드 | 타입 | 제약 |
|---|---|---|
| order_id | str | 존재해야 함, 현재 상태가 `CONFIRMED`여야 함 |

### 정상 흐름
1. 주문을 조회하고 연결된 시료를 조회한다.
2. 상태를 `CONFIRMED` → `RELEASE`로 전이한다.
3. 시료 재고를 주문 수량만큼 차감한다(`stock -= quantity`).

### 예외 케이스
- 존재하지 않는 `order_id` → `KeyError`
- `CONFIRMED`가 아닌 주문(`RECEIVED`/`PRODUCTION`/`REJECTED`/`RELEASE`)을 출고 시도 → `ValueError`
- (TODO) 재고보다 많은 수량이 이미 승인되어 있었던 경우(승인 이후 다른 경로로 재고가 줄어든 경우) 출고 시점에 재고가 음수가 될 수 있는지 → 현재 `update_stock`은 검증 없이 그대로 차감. 재고 음수 방지 로직 필요 여부 논의.
