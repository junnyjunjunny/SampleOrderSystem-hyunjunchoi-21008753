## 개요 (설계 방향 확정, 구현은 아직)

- `PRODUCTION` 상태 주문은 더 이상 수동으로 "생산 완료" 처리하지 않는다.
- 시료(`Sample`)의 `avg_production_time`(평균 생산시간)에 따라 시간이 지나면 자동으로 생산되고, 생산량이 필요량(주문 수량 - 접수 시점 재고 부족분)을 채우면 자동으로 `PRODUCTION → CONFIRMED`로 전이하며 재고를 보충한다.
- 이 메뉴는 그 자동 생산 진행 상황(어느 주문이 얼마나 생산됐는지, 언제쯤 완료될지)을 조회하는 화면이 될 예정이다.

## 논의 필요 항목

- [ ] 자동 생산 진행을 어떻게 계산할지: 실제 경과 시간(`datetime.now() - 생산 시작 시각`) 기반으로 매번 조회 시점에 계산할지, 별도 스케줄러/틱 방식이 필요한지
- [ ] `PRODUCTION` 진입 시각을 어디에 저장할지(`Order`에 필드 추가 필요 — 현재 `created_at`은 접수 시각이라 별도 컬럼 필요)
- [ ] "생산 라인"을 새 엔티티로 모델링할지(예: `ProductionLine(line_id, capacity, ...)`), 아니면 `Order`에 `line_id` 필드만 추가할지, 아니면 이번 자동 생산 설계에서는 "라인" 개념 없이 시료 단위 평균 생산시간만으로 계산할지
- [ ] `OrderService.complete_production`(현재 수동 API로 존재, CLI에서는 이제 호출되지 않음)을 이 자동 로직이 그대로 재사용할지, 새로 만들지
