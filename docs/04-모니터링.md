# 4. 모니터링

## 개요

메인 메뉴 `[4] 모니터링`. 주문량 확인과 재고량 확인 두 가지 하위 메뉴를 제공한다.

## 현재 구현 상태

- 주문량 확인(상태별 건수 + 목록): 구현됨 (`check_order_volume`)
- 재고량 확인(시료별 재고 잔여율): 구현됨 (`check_stock_level`)

## 하위 메뉴 구조

`[4] 모니터링` 진입 시:
```
[1] 주문량 확인  [2] 재고량 확인  [0] 뒤로 > 
```

### `[1]` 주문량 확인
1. `RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE` 각 상태별 건수를 먼저 보여준다. `REJECTED`는 이 화면에서 제외한다(건수·목록 모두).
2. 그 아래에 위 상태들에 해당하는 주문 전체를 표로 보여준다(주문ID/시료ID/고객/수량/상태).

### `[2]` 재고량 확인
등록된 모든 시료를 표로 보여준다. 컬럼: **시료명 / 재고 / 상태**.
"상태"는 고정 용량 대비 퍼센트가 아니라 **주문대비**(해당 시료를 참조하는 미해결 주문 수량 합 대비 재고)로 판정한다:
- 미해결 주문 수량 합 = 상태가 `RESERVED` 또는 `PRODUCING`인 주문의 `quantity` 합(아직 출고/거절되지 않아 재고에 부담을 주는 진행 중 수요).
- 재고 == 0 → `고갈`(빨간색) — 미해결 수요가 있든 없든 재고가 0이면 무조건 고갈.
- 재고 >= 미해결 주문 수량 합 → `여유`(초록색)
- 그 외(0 < 재고 < 미해결 주문 수량 합) → `부족`(주황색)
- 색상은 ANSI 이스케이프로 표시(여유=`\033[92m`, 부족=`\033[38;5;208m`, 고갈=`\033[91m`).
- 잔여율(%) 표시는 없다 — 분모(기준 용량)를 시료마다 정의할 근거가 없어 제거했다.

## 데이터/상태

기존 `Order`/`Sample` 조회 기능을 그대로 사용, 별도 집계 테이블/모델 없음. 미해결 수요 계산(`OrderService.pending_order_quantity`)과 여유/부족/고갈 판정(`stock_status`)은 `services.py`에 둔다.

## TDD 체크리스트 (테스트 대상)

콘솔 표시 함수(`check_order_volume`, `check_stock_level`)는 다른 화면 출력 함수와 동일하게 별도 유닛 테스트를 추가하지 않는다. 데이터 조회 자체는 기존 `OrderRepository.list_all`/`SampleRepository.list_all` 테스트로 커버된다.

## 완료 기준

`python main.py`로 직접 실행해 두 하위 메뉴가 요구사항대로 표시되는지 확인.

## 관련 파일

`app/cli.py`(`monitoring_menu`, `check_order_volume`, `check_stock_level`)
