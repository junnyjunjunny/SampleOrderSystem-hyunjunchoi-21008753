# 2. 시료 주문

## 개요

고객이 특정 시료를 특정 수량만큼 주문 접수하는 기능. 접수된 주문은 `RECEIVED` 상태로 시작하며, 이후 승인/거절(3번 메뉴)로 이어진다.

## 현재 구현 상태

- 주문 접수: 구현됨 (`receive_order`)

## 상세 요구사항

### 입력값
| 필드 | 타입 | 제약 |
|---|---|---|
| order_id | str | 필수, 중복 불가 |
| sample_id | str | 필수 |
| customer_name | str | 필수 |
| quantity | int | 1 이상 |

### 정상 흐름
1. 사용자가 주문 ID/시료 ID/고객명/수량을 입력한다.
2. `OrderRepository.create`로 상태 `RECEIVED`, 접수 시각(UTC ISO)과 함께 저장한다.

### 예외 케이스
- 이미 존재하는 `order_id`로 접수 시 → `ValueError`
